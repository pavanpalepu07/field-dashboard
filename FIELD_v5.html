Connect Live to Microsoft Forms Response Sheet

The dashboard reads your Microsoft Forms response Excel file directly using the Microsoft Graph API — no CORS issues, no manual downloads. Choose your connection method below.

⭐ Method 1 — Graph API (Recommended)

Connects directly to your SharePoint/OneDrive file. No CORS errors. Refreshes on demand. Requires a one-time Azure App Registration.

1
Register an app in Azure Portal → App Registrations. Set platform to Single-page application. Add http://localhost and your dashboard URL as redirect URIs.
2
Under API Permissions, add: Files.Read and Sites.Read.All (Microsoft Graph, Delegated). Click Grant admin consent.
3
Copy your Application (Client) ID from the app overview page.
4
In Microsoft Forms → Responses → Open in Excel. Copy the resulting SharePoint file URL from your browser address bar.
5
Paste both below and click Connect via Graph API.
🆓 Method 2 — Power Automate + SharePoint List (Free)

Instead of the premium Response action, the flow writes data to a SharePoint List. The dashboard reads from SharePoint via the Graph API — 100% free connectors only.

✓ Free connectors used: Microsoft Forms · SharePoint · Excel Online (Business)
✗ Premium connectors NOT needed: No HTTP trigger · No Response action
1
Create a SharePoint List named FIELD Governance Data in any SharePoint site you have access to. Add one column: Title (rename to ResponseJSON, type: Multiple lines of text).
2
In Power Automate → + Create → Automated cloud flow → Trigger: "When a new response is submitted" (Microsoft Forms) → Select your form → Create.
3
Add step: Microsoft Forms → "Get response details" → Form: your form → Response Id: dynamic content "List of response notifications Response Id".
4
Add step: SharePoint → "Create item" → Site: your SharePoint site → List: FIELD Governance Data → ResponseJSON field: click Expression tab → paste: string(body('Get_response_details')) → OK.
5
Save the flow. Now enter your SharePoint site URL and List name below and click Connect via SharePoint List. The dashboard uses Graph API to read the list — no CORS issues.
💡 Alternative — No flow at all: If managers submit via Microsoft Forms and you open the response Excel in OneDrive, just use Method 3 (Upload) before each call. It takes 30 seconds and requires zero setup.
📁 Method 3 — Manual Upload

Download the Excel from OneDrive and upload it directly. No setup required. Best for weekly use before governance calls.

1
In Microsoft Forms → Responses → Open in Excel. This opens the live-linked Excel in OneDrive.
2
Click File → Download a Copy in Excel Online (or use OneDrive → Download).
3
Click Upload Excel File below and select the downloaded file.
Why does direct link not work?
Browsers enforce CORS (Cross-Origin Resource Sharing) which blocks direct fetches to SharePoint/OneDrive URLs from a local HTML file. Methods 1 and 2 solve this permanently using Graph API — no CORS issues. Method 3 is the quickest workaround.
🔄
Auto-Refresh (Graph API & Power Automate only)
Once connected via Method 1 or 2, enable auto-refresh to pull fresh data on a schedule.

Weekly Governance Form — Field Reference Guide

This guide explains every field in the Microsoft Forms weekly submission. Use it to understand what data to enter, how to rate your team accurately, and what triggers an Amber or Red status on the dashboard.

35 Fields 6 Sections 🤖 RAG Auto-Calculated
🟢 GREEN — On Track
All targets met, no critical issues, team stable and performing well.
Targets ≥ 90% · Morale ≥ 3.5 · No critical incidents
🟡 AMBER — At Risk
Some targets missed, minor people issues, or a blocker under active management.
Targets 75–89% · 1–2 high risks · Morale 3.0–3.4
🔴 RED — Critical
Significant targets missed, resignation, unresolved blocker, or escalation needed urgently.
Targets < 75% · Any resignation · Blocker > 3 days unresolved
🤖
RAG Status is Automatically Calculated — No Need to Select It
The Overall Project Health (RAG) status is not a question in the form. It is computed automatically from your numeric responses each time the dashboard loads. The engine checks the following triggers:
🔴 RED — any one of:
Any resignation
Any client complaint
Any critical incident
3+ High Escalations
Morale below 3.0
Both targets & SLA below 75%
Headcount 15%+ below plan
3+ High Escalations (combined)
🟡 AMBER — 2+ of:
Targets 75–89%
SLA 75–89%
Morale 3.0–3.4
1–2 High Escalations
3+ Medium Issues
5–10 Open Issues
2+ Dependency Blockers
2+ Milestones at Risk
Training below 80%
🟢 GREEN — all clear:
Targets ≥ 90%
SLA ≥ 90%
Morale ≥ 3.5
No resignations
No complaints
No critical incidents
Fewer than 2 Amber signals
Headcount on plan
